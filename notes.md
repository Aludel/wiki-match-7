# 操作系统：信号量 (Semaphore)

> 这是一篇示例短笔记。完整整理见 [[信号量详解（OSTEP 第31章）]]。

## 一句话定义
信号量是一个**带整数值的对象**，配两个原子操作：sem_wait（值减 1，小于 0 就睡）和 sem_post（值加 1，有人睡就唤醒一个）。

## 关键恒等式
当信号量的值为负时，**它的绝对值 = 正在等待的线程数**。记住这条，大多数题目都能心算。

## 初值速查
| 用途 | 初值 | 含义 |
| --- | --- | --- |
| 作锁（互斥） | 1 | 一个名额 |
| 作条件变量 | 0 | 没名额，需等别人 post |
| 生产者 empty | MAX | 初始空位数 |
| 生产者 full | 0 | 初始没有数据 |

# 信号量详解（OSTEP 第 31 章）

> 本笔记由《信号量学习指南》整理而来，是一篇可长期复习的完整笔记。相关短笔记：[[操作系统：信号量]]。

## 0. 三个前置概念
- **线程**：把程序想成餐厅，线程就是同时干活的服务员。多个线程并发执行，效率高但会"撞车"。
- **临界区**：同一时刻最多只能一个线程进入的代码（如同只能容一人的储藏室）。需要锁来保护。
- **睡眠/唤醒**：等不到资源时，线程先去"睡觉"放弃 CPU，等别人 post 时再被唤醒，避免忙等浪费 CPU。

## 1. 信号量的定义
信号量是一个**带整数值的对象**，加两个原子操作。使用前必须初始化：

```
#include <semaphore.h>
sem_t s;
sem_init(&s, 0, 1);   // 第3个参数 1 是初始值，至关重要
```

- 第 2 个参数填 0：表示同一进程内多线程共享。
- 第 3 个参数：**初始值**，可理解为"现在还有几个名额/资源可用"。

## 2. wait / post 做了什么

```
sem_wait(s):  值 -1;  若值 < 0 则当前线程睡眠等待
sem_post(s):  值 +1;  若有线程在等待，唤醒其中一个
```

- sem_wait 要么立即返回（值 ≥ 0），要么把自己挂起睡眠。
- sem_post **从不阻塞**，只在有线程等待时才唤醒一个。

> **最重要的恒等式**：当信号量的值为负时，它的绝对值 = 正在排队等待的线程数。例如值为 -3，说明此刻有 3 个线程睡着了在等。记住这条，多数题目都能心算。

## 3. 二值信号量（用作锁）
初值填 **1**。锁只有"持有/空闲"两态，故称二值信号量。

```
sem_t m;
sem_init(&m, 0, 1);   // 初值 = 1
sem_wait(&m);         // 上锁
// 临界区
sem_post(&m);         // 解锁
```

第一个线程 wait 后值变 0（占住）；它在临界区时第二个线程来 wait 会变 -1 而睡着；等第一个线程 post 把值加回并唤醒它。若有 3 个线程排队等锁，信号量值会是 **-3**。

## 4. 用作条件变量（一个等一个）
初值填 **0**。典型场景：父线程等子线程跑完。

```
sem_init(&s, 0, 0);   // 初值 = 0
// 子线程: 做完后 sem_post(&s);
// 父线程: sem_wait(&s);  在这里等子线程
```

初值 0 意味着一开始没有名额，谁先 wait 谁就睡。无论"父先跑"还是"子先跑完"，结果都正确：

- 父先跑：值 0→-1，父睡眠 → 子运行后 post 值 -1→0，唤醒父 → 父继续。
- 子先跑完：值 0→1（没人等，不唤醒）→ 父后来 wait 值 1→0，不用睡，直接返回。

## 5. 生产者/消费者（有界缓冲区）
规则：缓冲区满了生产者等，空了消费者等。用两个信号量：empty（空位数，初值 MAX）、full（数据数，初值 0）。

**问题一 · 竞态**：MAX>1 且多生产者时，写数据和更新下标这段临界区没保护，会互相覆盖丢数据。

**问题二 · 死锁**：若把 mutex 放最外层（包住 empty/full 的 wait），消费者会抱着锁去等 full，生产者要锁才能造数据 → 循环等待 → 死锁。

**正确方案 · 缩小锁范围**：

```
// 生产者
sem_wait(&empty);   // 等空位（在锁外）
sem_wait(&mutex);   // 上锁
put(i);             // 临界区
sem_post(&mutex);   // 解锁
sem_post(&full);    // 通知有数据（在锁外）

// 消费者
sem_wait(&full);
sem_wait(&mutex);
tmp = get();
sem_post(&mutex);
sem_post(&empty);
// main: empty=MAX, full=0, mutex=1
```

> 口诀：**先等条件，再上锁；先解锁，再发信号。** 条件信号量（empty/full）在外圈，互斥锁（mutex）紧贴临界区在内圈。线程睡觉时手里不攥着锁，死锁就解开了。

## 6. 读者—写者锁
允许"多读者并发"或"一个写者独占"。核心机巧：**第一个读者代表大家抢下 writelock**，挡住写者；之后读者直接进；**最后一个读者离开时**才释放 writelock。

```
acquire_readlock:  lock; readers++; if(readers==1) wait(writelock); unlock;
release_readlock:  lock; readers--; if(readers==0) post(writelock); unlock;
acquire_writelock: wait(writelock);
release_writelock: post(writelock);
```

> 缺陷：只要源源不断有读者，写者可能**饿死**。而且读写锁较复杂、未必更快——Hill 定律："大而笨可能更好"，先尝试简单的普通锁。

## 7. 哲学家就餐
5 人围桌，每两人间一把叉，要同时拿到左右两把才能吃。叉子用 5 个初值为 1 的信号量表示。

**死锁场景**：若每人都先拿起左叉，就会人手一把、首尾相连成环、全部卡死。

**破除依赖**：让编号最大的 P4 反过来**先拿右叉再拿左叉**。只要一个人顺序不同，环就断了，死锁消失。这是"破除循环等待"反死锁策略的应用。

## 8. 用锁+条件变量实现信号量（Zemaphore）

```
typedef struct { int value; pthread_cond_t cond; pthread_mutex_t lock; } Zem_t;

Zem_wait(s):  lock; while(s->value <= 0) Cond_wait(&cond,&lock); s->value--; unlock;
Zem_post(s):  lock; s->value++; Cond_signal(&cond); unlock;
```

> 两个易错点：① 必须用 **while** 而非 if 重新检查条件（被唤醒后条件未必仍满足）。② 该实现里 value 永不为负（与现代 Linux 一致），不像原始定义用负值表示等待线程数。

## 9. 速查表
| 用途 | 初值 | 含义 |
| --- | --- | --- |
| 作锁（互斥） | 1 | 一个名额，谁进谁占 |
| 作条件变量 | 0 | 没名额，等别人 post |
| 生产者 empty | MAX | 缓冲区初始空位数 |
| 生产者 full | 0 | 初始没有数据 |
| 读写锁 writelock | 1 | 独占权，一次一个写者 |
| 餐叉 forks[i] | 1 | 每把叉一个名额 |

## 10. 五条核心原则
1. **恒等式**：信号量为负时，绝对值 = 正在等待的线程数。
2. **初值决定一切**：1 = 锁；0 = 等待；MAX = 资源池容量。
3. **wait 可能睡，post 从不睡**：post 只在有人等时才唤醒。
4. **反死锁**：条件信号量在外圈，互斥锁紧贴临界区在内圈；睡觉别攥着锁。
5. **破除循环等待**：哲学家问题靠改一个人的拿叉顺序打破环。
